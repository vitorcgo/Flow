# Criterios de pronto

## MVP pronto

O projeto esta pronto para portfolio quando:

- usuario consegue logar
- dashboard mostra metricas reais do seed
- tickets podem ser criados, filtrados, movidos e comentados
- auditoria registra acoes importantes
- relatorios exibem dados uteis
- API possui documentacao
- deploy esta acessivel
- README explica o projeto sem enrolacao

## Qualidade minima

- Sem dados sensiveis no repositorio.
- Sem pastas de ferramenta.
- Sem arquivos temporarios.
- Sem commit criado sem aprovacao do dono.
- Comentarios de codigo em portugues.
- Nomes de dominio em portugues: classes, ids, variaveis, tabelas, colunas e componentes.
- Sem imagens pesadas sem necessidade.
- Sem rotas quebradas.
- Sem tela vazia em demo.

## Backend

- Controllers pequenos.
- Services com regra clara.
- Exceptions centralizadas.
- Status codes consistentes.
- Validacao de input.
- Logs com request id.

## Frontend

- Interface corporativa, limpa e densa.
- Componentes reutilizaveis.
- Estados de loading, vazio e erro.
- Responsivo em mobile.
- Acessivel por teclado nos fluxos principais.

## Portfolio

- Case honesto.
- Prints grandes e legiveis.
- Stack visivel.
- Problema, solucao e resultado bem explicados.
- Link do repositorio e preview funcionando.
