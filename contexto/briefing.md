# Briefing

## Ideia

FlowDesk e uma plataforma interna para empresas acompanharem chamados, tarefas operacionais e SLA.

O sistema substitui planilhas e mensagens soltas por um fluxo rastreavel:

- solicitacao entra como ticket
- equipe classifica prioridade
- responsavel assume
- SLA comeca a contar
- ticket anda pelo Kanban
- comentarios registram contexto
- auditoria guarda mudancas importantes
- gestor acompanha indicadores

## Publico

- Recrutadores tecnicos
- Tech leads
- Gestores de produto
- Empresas com suporte, financeiro, TI, operacao ou atendimento

## Persona principal

Nome: Marina, coordenadora de operacoes.

Problema:

- Recebe demandas por WhatsApp, e-mail e planilha.
- Nao sabe quem esta fazendo o que.
- Nao tem historico confiavel.
- Prazo estoura sem alerta.

Valor do FlowDesk:

- centraliza tickets
- define responsaveis
- mostra SLA
- registra auditoria
- gera relatorios simples

## Persona tecnica

O que ele quer ver:

- permissoes bem definidas
- banco normalizado
- testes
- logs uteis
- deploy reproduzivel
- separacao limpa entre frontend/backend

## Posicionamento no portfolio

Texto curto:

> Sistema corporativo de chamados, Kanban e SLA com Java Spring Boot, React TypeScript, PostgreSQL, autenticação JWT, RBAC, auditoria e deploy.

Texto de impacto:

> Um projeto pensado para parecer ferramenta interna real: menos landing page, mais fluxo de empresa, permissao, prazo, historico e operacao.
